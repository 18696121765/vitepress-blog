## 你好，我是王家鑫

我来自湖北襄阳，软件技术专业背景，目前是一名前端开发工程师，拥有 **4 年前端开发经验**。过去主要做 **Vue / uni-app** 相关业务：从 0 到 1 搭建、持续迭代、联调上线与性能体验优化，并能和产品、UI、后端、测试高效协作，保证交付与质量。

我更关注这些事：

- **可维护性**：统一封装（请求/登录态/上传/权限），减少重复与隐性耦合
- **体验一致性**：对复杂流程做兜底（登录态失效、身份跳转、异常提示），降低用户与客服成本
- **交付确定性**：拆分模块与排期可视化，及时同步风险点，按节点可交付

## 项目经验精选

### 百工聚 APP（2025.06 - 2025.10｜uni-app / uview-plus / Vue3）

由「中建三局 i 友通」微信小程序重构为 APP，面向建筑产业工人提供实名制、考勤、招工找活等一站式服务，并新增道具、商城等能力。我主要负责除工分外的大部分核心模块。

我在这个项目里重点做了几件“难但值”的事：

- **登录链路兜底**：接入阿里云一键登录 SDK，封装统一环境检测与降级策略（无流量/不支持时自动走验证码），并统一处理 401 失效、主动退出后的重判定，减少重复判断与分支
- **多身份体系**：工人/分包/总包按业务优先级动态判定与跳转，避免身份冲突导致的错端问题
- **实名与合同**：对接腾讯电子签；合同包含多类型（电子签链接 + PDF 签署），针对 PDF 预览与签名场景采用 **独立 H5（pdf.js）+ WebView** 的方案降低包体积与加载压力
- **招工找活 + 道具体系**：支持发布与联系（会员/道具扣减），不足时引导到充值页，保证闭环清晰
- **上架合规**：完善隐私协议与权限说明（定位/相机等）及 `manifest.json` 配置，降低上架驳回风险

### 省有道小程序（2025.02 - 2025.05｜uni-app / uview）

展会舞台/音响/桁架/LED 等租赁服务平台，覆盖全国市区。

- **登录**：基于微信授权手机号登录（`getPhoneNumber` -> 服务端换取手机号 -> 登录落地）
- **商品链路**：租赁/购买模式统一下单流程（按 `type` 分支），先生成订单与总价再拉起 `uni.requestPayment`
- **定位**：对接腾讯地图（坐标转换、逆地址解析等），控制调用频率与配额，避免“下拉刷新反复定位”造成的体验抖动与配额不足

### 盛宁智享园区服务平台（2024.08 - 2025.01｜uni-app / uview / ruoyi）

园区缴费查询、会议室预约、职位与需求发布等一站式服务。

- **支付链路**：对接富友支付，采用“预订单 -> 获取 `orderinfo` -> 正式支付”的链路，便于排查与对账
- **上传组件二次封装**：对 `u-upload` 做二次封装：统一基地址、自动携带 token、无 token 直接拦截、统一抛出 URL，减少页面逻辑与重复代码
- **审核合规**：梳理定位/资讯类目等审核材料与权限申请要点，降低审核失败概率

### 企业内部后台管理系统（2024.04 - 2024.08｜Vue / Vue-cli / Vue Router / Vuex / Axios）

企业架构、人员分配、社保管理、问卷反馈等管理系统。

- **登录与权限**：ElementUI 表单校验 + token 存储；通过路由守卫控制访问权限
- **按钮级权限**：封装统一权限校验方法/指令，按功能标识判断显示/禁用，减少散落 if 判断
- **头像上传**：二次封装 `el-upload`，统一请求头与返回值，配合表单校验提升体验

### 福移智慧商铺小程序（2023.06 - 2024.03｜uni-app / uview）

基于聚合支付底座搭建 SaaS 应用平台。我负责从 0 搭建小程序并开发登录、安防、工作号等模块。

- **从 0 搭建**：完成目录结构、请求封装与基础页面框架，保证后续多人协作可扩展
- **二维码兼容**：解决真机端 base64 二维码 canvas 无法显示/保存问题，采用文件系统落盘再展示/保存的方案
- **互斥业务**：实体卡/虚拟卡互斥逻辑，通过状态查询接口判定并跳转正确订购路径，减少误操作
